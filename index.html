Git & GitHub'ın Önemi

  • Versiyon Kontrolü: Kodun geçmişini saklar, yapılan her değişikliğe istediğin zaman geri dönebilirsin.
  • Dağıtık Yapı: İnternete bağlı olmasan bile yerel depoda çalışabilir, sonrasında senkronize edebilirsin.
  • Hata Yönetimi: Yanlış bir değişiklik yaptığında “commit geçmişi” sayesinde kolayca önceki sürüme dönebilirsin.
  • Branch (Dal) Kullanımı: Yeni özellikleri veya denemeleri ana koda zarar vermeden ayrı dallarda geliştirebilirsin.
  • Takım Çalışmasını Kolaylaştırma: Birden fazla geliştirici aynı projede paralel çalışabilir, değişiklikler birleştirilebilir.
  • Bulut Depolama: GitHub, projelerini bulutta saklamanı ve istediğin yerden erişmeni sağlar.
  • Açık Kaynak Katkısı: Dünya çapında açık kaynak projelere katkı yapabilirsin, portföyüne güçlü referans ekleyebilirsin.
  • İşbirliği Kolaylığı: “Pull Request”, “Issues” gibi özellikler sayesinde ekip içinde kod inceleme ve görev takibi yapılır.
  • Otomasyon ve CI/CD: GitHub Actions gibi özelliklerle testleri, deploy süreçlerini otomatikleştirebilirsin.
  • Profesyonel Görünüm: GitHub profili, iş başvurularında bir nevi portföy olarak görülür ve kod kaliteni gösterir.
  • Git: Kodunu bilgisayarında versiyon kontrol eder.
  • GitHub: Bu depoları internet üzerinden paylaşmanı ve işbirliği yapmanı sağlar.
  • Tekrar tekrar git init yapmak remote ayarlarını bozabilir.
  • Bir proje klasöründe sadece ilk defa yapılır.
  • git add . → dosyaları staging alanına alır.
  • git commit -m "mesaj" → o anki halini kayda alır.
  • Ana dal (main / master) genelde stabil kod içindir.
  • Yeni özellik/deneme için ayrı branch aç (git checkout -b feature-x).
  • Merge veya Pull Request ile ana dala alınır.
  • Commit küçük, sık ve anlamlı olmalı
